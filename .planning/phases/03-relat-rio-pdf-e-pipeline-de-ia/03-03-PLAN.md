    Create worker/processors/ai-describe-item.ts exporting async function processAiDescribeItem(job: Job&lt;AiDescribeItemJob&gt;): Promise&lt;void&gt;.

    Algorithm (must be linear two-stage, no agent framework):
    1. Read job.data.itemId and vistoriaId. Load Item with midias via prisma.item.findUnique; throw UnrecoverableError or regular Error if item missing.
    2. Set item.status = EM_ANALISE.
    3. Select latest AUDIO midia and latest FOTO midia. If either missing, set status back to CAPTURADO (or leave CAPTURADO) and return early without throwing (not ready).
    4. Load file bytes:
       - Prefer reading local file via resolveLocalUploadPath(midia.url) if file exists under public/.
       - Else if storage key present, use getStorageProvider to fetch (getSignedUrl + fetch, or GetObject if implemented). If GetObject not on interface, add a download(key): Promise&lt;Buffer&gt; method to StorageProvider in this task (allowed small extension to Plan 01 types) OR fetch signed URL.
    5. Stage 1 STT: call AIRouter.transcribeAudio on audio buffer with mime audio/webm or audio/mpeg based on extension. Save transcript on audio Midia.transcricao.
    6. Stage 2: call AIRouter.generateDescription(photoBuffer, image mime, transcript).
    7. Persist AnaliseIa: provedor mapped from meta.provider (GEMINI|OPENAI|CLAUDE), input: { transcript, photoKey, promptsVersion: "phase3-v1" }, output: raw data JSON, analise: technicalDescription, custo/tokens if available, latencyMs, usedFallback, guardrailHit, stage: "full".
    8. Update Item: descricao = technicalDescription, descricaoFinal = technicalDescription (editable baseline), estadoConservacao = mapItemStateToPrisma(itemState), status = ANALISADO, analisadoEm = now(), descricaoEditada = false.
    9. On provider failure: rethrow to let BullMQ retry; do not mark ANALISADO. After final failure BullMQ will exhaust attempts — optionally set a note in item.descricao null and status CAPTURADO with console.error (document choice in SUMMARY).
    10. Never write the words laudo/perícia into prompts beyond the ban instruction; never store un-sanitized regulated terms (router already enforces).

    Do not generate PDF here.

## Trust Boundaries

| Worker → AI providers | Media bytes sent to third parties |
| Worker → PostgreSQL | Generated text written as item descriptions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Spoofing | Media upload API | mitigate | Require NextAuth session; bind itemId to vistoriaId path param |
| T-03-10 | Tampering | Job payload itemId | mitigate | Worker reloads Item/Midia from Prisma; ignores client-supplied file paths in job |
| T-03-11 | Information Disclosure | Worker logs | mitigate | Log ids/status only; never log full transcripts in production info level if they contain PII addresses — allow transcript in AnaliseIa.input for audit |
| T-03-12 | Denial of Service | AI fan-out | mitigate | BullMQ concurrency 2; 3 attempts with exponential backoff |
| T-03-13 | Elevation of Privilege | Cross-tenant item | mitigate | Validate item.ambiente.vistoriaId === job.vistoriaId before processing |
| T-03-SC | Tampering | npm installs | accept | No new packages in this plan |
